   54 -- | An implementation of the __Random-Improve__ coin selection algorithm.
   55 --
   56 -- = Overview
   57 --
   58 -- The __Random-Improve__ coin selection algorithm works in __two phases__, by
   59 -- /first/ selecting UTxO entries /at random/ to pay for each of the given
   60 -- outputs, and /then/ attempting to /improve/ upon each of the selections.
   61 --
   62 -- === Phase 1: Random Selection
   63 --
   64 -- __In this phase, the algorithm randomly selects a minimal set of UTxO__
   65 -- __entries to pay for each of the given outputs.__
   66 --
   67 -- During this phase, the algorithm:
   68 --
   69 --   *  processes outputs in /descending order of coin value/.
   70 --
   71 --   *  maintains a /remaining UTxO set/, initially equal to the given
   72 --      /UTxO set/ parameter.
   73 --
   74 -- For each output of value __/v/__, the algorithm /randomly/ selects entries
   75 -- from the /remaining UTxO set/, until the total value of selected entries is
   76 -- greater than or equal to __/v/__. The selected entries are then associated
   77 -- with that output, and removed from the /remaining UTxO set/.
   78 --
   79 -- This phase ends when every output has been associated with a selection of
   80 -- UTxO entries.
   81 --
   82 -- However, if the remaining UTxO set is completely exhausted before all
   83 -- outputs can be processed, the algorithm terminates with an error.
   84 --
   85 -- === Phase 2: Improvement
   86 --
   87 -- __In this phase, the algorithm attempts to improve upon each of the UTxO__
   88 -- __selections made in the previous phase, by conservatively expanding the__
   89 -- __selection made for each output.__
   90 --
   91 -- During this phase, the algorithm:
   92 --
   93 --   *  processes outputs in /ascending order of coin value/.
   94 --
   95 --   *  continues to maintain the /remaining UTxO set/ produced by the previous
   96 --      phase.
   97 --
   98 --   *  maintains an /accumulated coin selection/, which is initially /empty/.
   99 --
  100 -- For each output of value __/v/__, the algorithm:
  101 --
  102 --  1.  __Calculates a /target range/__ for the total value of inputs used to
  103 --      pay for that output, defined by the triplet:
  104 --
  105 --      (/minimum/, /ideal/, /maximum/) = (/v/, /2v/, /3v/)
  106 --
  107 --  2.  __Attempts to /improve/ upon the /existing UTxO selection/__ for that
  108 --      output, by repeatedly selecting additional entries at random from the
  109 --      /remaining UTxO set/, stopping when the selection can be improved upon
  110 --      no further.
  111 --
  112 --      A selection with value /v1/ is considered to be an /improvement/ over a
  113 --      selection with value /v0/ if __all__ of the following conditions are
  114 --      satisfied:
  115 --
  116 --       * __Condition 1__: we have moved closer to the /ideal/ value:
  117 --
  118 --             abs (/ideal/ − /v1/) < abs (/ideal/ − /v0/)
  119 --
  120 --       * __Condition 2__: we have not exceeded the /maximum/ value:
  121 --
  122 --             /v1/ ≤ /maximum/
  123 --
  124 --       * __Condition 3__: when counting cumulatively across all outputs
  125 --       considered so far, we have not selected more than the /maximum/ number
  126 --       of UTxO entries specified by 'limit'.
  127 --
  128 --  3.  __Creates a /change value/__ for the output, equal to the total value
  129 --      of the /final UTxO selection/ for that output minus the value /v/ of
  130 --      that output.
  131 --
  132 --  4.  __Updates the /accumulated coin selection/__:
  133 --
  134 --       * Adds the /output/ to 'outputs'.
  135 --       * Adds the /improved UTxO selection/ to 'inputs'.
  136 --       * Adds the /change value/ to 'change'.
  137 --
  138 -- This phase ends when every output has been processed, __or__ when the
  139 -- /remaining UTxO set/ has been exhausted, whichever occurs sooner.
  140 --
  141 -- = Termination
  142 --
  143 -- When both phases are complete, the algorithm terminates.
  144 --
  145 -- The /accumulated coin selection/ and /remaining UTxO set/ are returned to
  146 -- the caller.
  147 --
  148 -- === Failure Modes
  149 --
  150 -- The algorithm terminates with an __error__ if:
  151 --
  152 --  1.  The /total value/ of the initial UTxO set (the amount of money
  153 --      /available/) is /less than/ the total value of the output list (the
  154 --      amount of money /required/).
  155 --
  156 --      See: __'InputValueInsufficientError'__.
  157 --
  158 --  2.  The /number/ of entries in the initial UTxO set is /smaller than/ the
  159 --      number of requested outputs.
  160 --
  161 --      Due to the nature of the algorithm, /at least one/ UTxO entry is
  162 --      required /for each/ output.
  163 --
  164 --      See: __'InputCountInsufficientError'__.
  165 --
  166 --  3.  Due to the particular /distribution/ of values within the initial UTxO
  167 --      set, the algorithm depletes all entries from the UTxO set /before/ it
  168 --      is able to pay for all requested outputs.
  169 --
  170 --      See: __'InputsExhaustedError'__.
  171 --
  172 --  4.  The /number/ of UTxO entries needed to pay for the requested outputs
  173 --      would /exceed/ the upper limit specified by 'limit'.
  174 --
  175 --      See: __'InputLimitExceededError'__.
  176 --
  177 -- = Motivating Principles
  178 --
  179 -- There are several motivating principles behind the design of the algorithm.
  180 --
  181 -- === Principle 1: Dust Management
  182 --
  183 -- The probability that random selection will choose dust entries from a UTxO
  184 -- set increases with the proportion of dust in the set.
  185 --
  186 -- Therefore, for a UTxO set with a large amount of dust, there's a high
  187 -- probability that a random subset will include a large amount of dust.
  188 --
  189 -- === Principle 2: Change Management
  190 --
  191 -- Ideally, coin selection algorithms should, over time, create a UTxO set that
  192 -- has /useful/ outputs: outputs that will allow us to process future payments
  193 -- with a minimum number of inputs.
  194 --
  195 -- If for each payment request of value __/v/__ we create a change output of
  196 -- /roughly/ the same value __/v/__, then we will end up with a distribution of
  197 -- change values that matches the typical value distribution of payment
  198 -- requests.
  199 --
  200 -- === Principle 3: Performance Management
  201 --
  202 -- Searching the UTxO set for additional entries to improve our change outputs
  203 -- is /only/ useful if the UTxO set contains entries that are sufficiently
  204 -- small enough. But it is precisely when the UTxO set contains many small
  205 -- entries that it is less likely for a randomly-chosen UTxO entry to push the
  206 -- total above the upper bound.
  207 --
  208 -- @since 1.0.0
  209 randomImprove
  210     :: (Ord i, Ord o, MonadRandom m)
  211     => CoinSelectionAlgorithm i o m
  212 randomImprove = CoinSelectionAlgorithm payForOutputs
  213 
  214 payForOutputs
  215     :: (Ord i, Ord o, MonadRandom m)
  216     => CoinSelectionParameters i o
  217     -> ExceptT CoinSelectionError m (CoinSelectionResult i o)
  218 payForOutputs params = do
  219     mRandomSelections <- lift $ runMaybeT $ foldM makeRandomSelection
  220         (inputCountMax, inputsAvailable params, []) outputsDescending
  221     case mRandomSelections of
  222         Just (inputCountRemaining, utxoRemaining, randomSelections) -> do
  223             (_, finalSelection, utxoRemaining') <- lift $ foldM
  224                 improveSelection
  225                     (inputCountRemaining, mempty, utxoRemaining)
  226                     (reverse randomSelections)
  227             pure $ CoinSelectionResult finalSelection utxoRemaining'
  228         Nothing ->
  229             throwE errorCondition
  230   where
  231     errorCondition
  232       | amountAvailable < amountRequested =
  233           InputValueInsufficient $
  234               InputValueInsufficientError
  235                   amountAvailable amountRequested
  236       | utxoCount < outputCount =
  237           InputCountInsufficient $
  238               InputCountInsufficientError
  239                   utxoCount outputCount
  240       | utxoCount <= fromIntegral inputCountMax =
  241           InputsExhausted
  242               InputsExhaustedError
  243       | otherwise =
  244           InputLimitExceeded $
  245               InputLimitExceededError $
  246                   fromIntegral inputCountMax
  247     amountAvailable =
  248         coinMapValue $ inputsAvailable params
  249     amountRequested =
  250         coinMapValue $ outputsRequested params
  251     inputCountMax =
  252         fromIntegral $ calculateLimit (limit params) $ fromIntegral outputCount
  253     outputCount =
  254         fromIntegral $ length $ coinMapToList $ outputsRequested params
  255     outputsDescending =
  256         L.sortOn (Down . entryValue) $ coinMapToList $ outputsRequested params
  257     utxoCount =
  258         fromIntegral $ L.length $ coinMapToList $ inputsAvailable params
  259 
  260 -- | Randomly select entries from the given UTxO set, until the total value of
  261 --   selected entries is greater than or equal to the given output value.
  262 --
  263 -- Once a random selection has been made that meets the above criterion, this
  264 -- function returns that selection as is, making no attempt to improve upon
  265 -- the selection in any way.
  266 --
  267 makeRandomSelection
  268     :: forall i o m . MonadRandom m
  269     => (Integer, CoinMap i, [([CoinMapEntry i], CoinMapEntry o)])
  270     -> CoinMapEntry o
  271     -> MaybeT m (Integer, CoinMap i, [([CoinMapEntry i], CoinMapEntry o)])
  272 makeRandomSelection
  273     (inputCountRemaining, utxoRemaining, existingSelections) txout = do
  274         (utxoSelected, utxoRemaining') <- coverRandomly ([], utxoRemaining)
  275         return
  276             ( inputCountRemaining - fromIntegral (L.length utxoSelected)
  277             , utxoRemaining'
  278             , (utxoSelected, txout) : existingSelections
  279             )
  280   where
  281     coverRandomly
  282         :: ([CoinMapEntry i], CoinMap i)
  283         -> MaybeT m ([CoinMapEntry i], CoinMap i)
  284     coverRandomly (selected, remaining)
  285         | L.length selected > fromIntegral inputCountRemaining =
  286             MaybeT $ return Nothing
  287         | sumEntries selected >= targetMin (mkTargetRange txout) =
  288             MaybeT $ return $ Just (selected, remaining)
  289         | otherwise =
  290             MaybeT (coinMapRandomEntry remaining) >>= \(picked, remaining') ->
  291                 coverRandomly (picked : selected, remaining')

  293 -- | Perform an improvement to random selection on a given output.
  294 improveSelection
  295     :: forall i o m . (MonadRandom m, Ord i, Ord o)
  296     => (Integer, CoinSelection i o, CoinMap i)
  297     -> ([CoinMapEntry i], CoinMapEntry o)
  298     -> m (Integer, CoinSelection i o, CoinMap i)
  299 improveSelection (maxN0, selection, utxo0) (inps0, txout) = do
  300     (maxN, inps, utxo) <- improve (maxN0, inps0, utxo0)
  301     return
  302         ( maxN
  303         , selection <> CoinSelection
  304             { inputs = coinMapFromList inps
  305             , outputs = coinMapFromList [txout]
  306             , change = mkChange txout inps
  307             }
  308         , utxo
  309         )
  310   where
  311     target = mkTargetRange txout
  312 
  313     improve
  314         :: (Integer, [CoinMapEntry i], CoinMap i)
  315         -> m (Integer, [CoinMapEntry i], CoinMap i)
  316     improve (maxN, inps, utxo)
  317         | maxN >= 1 && sumEntries inps < targetAim target = do
  318             coinMapRandomEntry utxo >>= \case
  319                 Nothing ->
  320                     return (maxN, inps, utxo)
  321                 Just (io, utxo') | isImprovement io inps -> do
  322                     let inps' = io : inps
  323                     let maxN' = maxN - 1
  324                     improve (maxN', inps', utxo')
  325                 Just _ ->
  326                     return (maxN, inps, utxo)
  327         | otherwise =
  328             return (maxN, inps, utxo)
  329 
  330     isImprovement :: CoinMapEntry i -> [CoinMapEntry i] -> Bool
  331     isImprovement io selected =
  332         let
  333             condA = -- (a) It doesn’t exceed a specified upper limit.
  334                 sumEntries (io : selected) < targetMax target
  335 
  336             condB = -- (b) Addition gets us closer to the ideal change
  337                 distanceA < distanceB
  338               where
  339                 distanceA = C.distance
  340                     (targetAim target)
  341                     (sumEntries (io : selected))
  342                 distanceB = C.distance
  343                     (targetAim target)
  344                     (sumEntries selected)
  345 
  346             -- (c) Doesn't exceed maximum number of inputs
  347             -- Guaranteed by the precondition on 'improve'.
  348         in
  349             condA && condB

  355 -- | Represents a target range of /total input values/ for a given output.
  356 --
  357 -- In this context, /total input value/ refers to the total value of a set of
  358 -- inputs selected to pay for a given output.
  359 --
  360 data TargetRange = TargetRange
  361     { targetMin :: Coin
  362         -- ^ The minimum value, corresponding to exactly the requested target
  363         -- amount, and a change amount of zero.
  364     , targetAim :: Coin
  365         -- ^ The ideal value, corresponding to exactly twice the requested
  366         -- target amount, and a change amount equal to the requested amount.
  367     , targetMax :: Coin
  368         -- ^ The maximum value, corresponding to exactly three times the
  369         -- requested amount, and a change amount equal to twice the requested
  370         -- amount.
  371     }
  372 
  373 -- | Compute the target range of /total input values/ for a given output.
  374 --
  375 -- See 'TargetRange'.
  376 --
  377 mkTargetRange :: CoinMapEntry o -> TargetRange
  378 mkTargetRange (CoinMapEntry _ c) = TargetRange
  379     { targetMin = c
  380     , targetAim = c `C.add` c
  381     , targetMax = c `C.add` c `C.add` c
  382     }
  383 
  384 -- | Compute change outputs from a target output and a selection of inputs.
  385 --
  386 -- Pre-condition:
  387 --
  388 -- The output must be less than (or equal to) the sum of the inputs.
  389 --
  390 mkChange :: CoinMapEntry o -> [CoinMapEntry i] -> [Coin]
  391 mkChange (CoinMapEntry _ out) inps =
  392     case difference of
  393         Nothing ->
  394             error $ mconcat
  395                 [ "mkChange: "
  396                 , "output must be less than or equal to sum of inputs"
  397                 ]
  398         Just d | C.isZero d ->
  399             []
  400         Just d ->
  401             [d]
  402   where
  403     difference = sumEntries inps `C.sub` out
